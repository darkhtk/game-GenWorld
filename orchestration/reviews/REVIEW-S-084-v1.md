## 페르소나 리뷰

### 🎮 캐주얼 게이머 (게이머)
#### 인상
월드 이벤트가 끝났는데 몬스터가 계속 남아있으면 혼란스럽다. 특히 "Blood Moon이 끝났다"고 알림이 뜨는데 강한 몬스터가 계속 돌아다니면 이상하다.

#### 문제점
- 이벤트 종료 후에도 특별 몬스터들이 그대로 남아있음
- 플레이어가 "이벤트가 정말 끝났나?" 의심하게 됨
- 게임 세계의 일관성이 떨어짐

#### 제안
- 이벤트 종료와 동시에 관련 오브젝트들이 사라져야 함
- 부드러운 페이드아웃 이펙트로 자연스럽게 제거

### ⚔️ 코어 게이머 (분석가)
#### 인상
시스템적으로 불완전한 상태. WorldEventEndEvent가 방출되지만 아무도 처리하지 않는 것은 아키텍처 설계 미완성을 의미한다.

#### 문제점
- invasion/elite_spawn/merchant 타입의 시작/종료 로직이 전혀 구현되지 않음
- 메모리 누수 가능성: 이벤트로 생성된 오브젝트들이 누적됨
- 게임 밸런스 파괴: Blood Moon 엘리트들이 영구히 남아 경험치 파밍 가능

#### 제안
- 각 이벤트 타입별로 정리 로직 구현 필요
- MonsterSpawner와 연계하여 이벤트별 몬스터 추적
- 이벤트 종료 시 해당 몬스터들만 선별적 제거

### 🎨 UX/UI 디자이너 (디자이너)
#### 인상
기능적으로는 UI 변경이 없지만, 플레이어 경험 측면에서 이벤트 종료 피드백이 불완전하다.

#### 문제점
- 이벤트 종료 알림과 실제 상태가 불일치
- 플레이어가 혼란을 느낄 수 있는 UX 결함
- 월드 상태의 시각적 피드백 부족

#### 제안
- 이벤트 종료 시 시각적/청각적 피드백 강화
- 남은 오브젝트들이 서서히 사라지는 연출 필요

### 🔍 QA 엔지니어 (테스터)
#### 인상
크리티컬한 메모리 누수와 게임플레이 일관성 버그가 예상되는 미완성 구현.

#### 문제점
- 이벤트 반복 시 오브젝트 누적으로 메모리 사용량 증가
- 씬 전환 시에도 이벤트 오브젝트들이 남을 수 있음
- WorldEventEndEvent 구독자 없어서 정리 로직이 실행되지 않음
- 예외 상황(강제 종료, 씬 전환 등)에서 정리 보장 안 됨

#### 제안
- 각 이벤트 타입별 정리 로직 구현 필수
- OnDestroy/OnSceneUnload에서도 정리 보장
- 이벤트 오브젝트 추적을 위한 태깅 시스템 필요

## 페르소나 회의록

### 쟁점
S-084 태스크가 아직 구현되지 않은 상태에서 리뷰하는 것이 적절한가?

### 토론
**🎮 캐주얼 게이머:** 구현이 안 되어 있다면 더더욱 문제다. 플레이어가 겪을 버그를 미리 막아야 한다.

**⚔️ 코어 게이머:** 맞다. 현재 상태로는 WorldEvent 시스템이 제대로 작동하지 않는다. 특히 invasion 타입은 정의만 있고 실제 로직이 없어서 아무 일도 일어나지 않는다.

**🎨 UX/UI 디자이너:** 사용자 관점에서 보면 "이벤트 시작/종료" 알림만 뜨고 실제로는 아무 변화가 없는 것과 같다. 이는 심각한 UX 문제다.

**🔍 QA 엔지니어:** 단순히 정리만 하는 것이 아니라 전체 이벤트 시스템의 완성도를 높여야 한다. bonus_drops만 구현되고 나머지는 비어있는 상태.

### 합의
모든 페르소나가 **NEEDS_WORK**에 동의. 단순한 오브젝트 정리를 넘어서 전체 WorldEvent 시스템의 완성이 필요하다는 공통된 의견.
