## 🔍 검증 결과

### ✅ 검증 1: 엔진 검증
- **씬 구조**: MainMenuScene, GameScene 존재 확인
- **UIManager**: 씬에 올바르게 배치되어 있음
- **컴포넌트 참조**: UIManager의 10개 UI 패널 참조 정상

### ❌ 검증 2: 코드 추적
**SPEC 요구사항 대비 구현 상태:**

| 요구사항 | 상태 | 비고 |
|---------|------|------|
| R1: HideAll on Scene Exit | ❌ **미구현** | GameUIWiring.cs:198에 호출 없음 |
| R2: sceneLoaded Hook | ❌ **미구현** | UIManager에 이벤트 구독 로직 없음 |
| R3: Dialogue State Reset | ✅ 구현됨 | HideAll()에서 _dialogueOpen = false |

**문제점:**
- `GameUIWiring.cs:198`에서 `SceneManager.LoadScene("MainMenuScene")` 전에 `UIManager.HideAll()` 호출이 없음
- `MainMenuController.cs:68,75`에서도 씬 로드 전 패널 정리 없음  
- UIManager에 SceneManager.sceneLoaded 이벤트 구독 로직 누락

### ❌ 검증 3: UI 추적
**현재 플로우 문제:**
1. 패널 열림 → ESC 메뉴 → "메인메뉴로" 클릭
2. `GameUIWiring.cs:198`에서 즉시 씬 전환
3. **열린 패널 상태가 그대로 다음 씬으로 누수**

### ❌ 검증 4: 플레이 시나리오
**시나리오 1: 인벤토리 열린 상태에서 메인메뉴 이동**
- 결과: 인벤토리가 메인메뉴에서도 열린 상태로 남음 (stale references)

**시나리오 2: 대화 중 메뉴 이동**  
- 결과: `_dialogueOpen` 플래그가 리셋되지 않아 입력 차단 지속

**시나리오 3: 메인메뉴에서 게임 재진입**
- 결과: 이전 패널 상태가 다시 나타날 수 있음

## 🎭 페르소나별 리뷰

### 🎮 캐주얼 게이머 관점
**우려사항:**
- 메인메뉴로 나갔다가 게임 재진입할 때 예상치 못한 화면이 나타날 수 있음
- 특히 인벤토리나 스킬트리가 갑자기 열려있으면 혼란스러움

**의견:** 
게임 플레이 흐름이 끊기는 느낌이 들 것 같다. 씬 전환할 때는 깔끔하게 모든 UI가 닫혀야 자연스럽다.

### ⚔️ 코어 게이머 관점
**기술적 분석:**
- stale reference 문제는 런타임에 크래시나 예상치 못한 동작을 일으킬 수 있음
- UI 상태 관리가 게임의 전반적인 안정성에 영향을 미침

**의견:**
UI 상태가 씬 간에 누수되는 것은 심각한 아키텍처 결함이다. 특히 인벤토리 같은 게임 데이터와 연결된 UI에서는 더욱 위험하다.

### 🎨 UX/UI 디자이너 관점
**사용자 경험 문제:**
- 일관성 부족: 씬 전환 후 예상치 못한 UI 상태
- 시각적 혼란: 메인메뉴에 게임 UI가 겹쳐 보일 수 있음

**의견:**
UI 상태 전환에 명확한 규칙이 필요하다. 씬 전환은 사용자에게 '새로운 컨텍스트'라는 신호를 주는데, 이전 UI가 남아있으면 멘탈 모델이 깨진다.

### 🔍 QA 엔지니어 관점  
**발견된 버그:**
1. **Critical**: SPEC R1, R2 요구사항 미구현
2. **High**: 씬 전환 시 UI 상태 누수로 인한 잠재적 크래시 위험
3. **Medium**: 입력 차단 상태가 씬 간에 지속될 수 있음

**테스트 케이스 실패:**
- 인벤토리 열고 메인메뉴 → 게임 재진입 시 UI 상태 확인
- 대화 중 강제 씬 전환 → 입력 차단 해제 확인
